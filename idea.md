# Library Management System

## 1. Project Overview

The Library Management System is a full-stack web application that automates library operations such as managing books, users, issuing and returning books, and fine calculation.

The system focuses heavily on backend architecture following software engineering and system design principles.

Backend Weightage: 75%
Frontend Weightage: 25%

---

## 2. Scope of the Project

The system supports:

- User Registration & Login
- Role-based Access (Admin, Member)
- Book Management (CRUD)
- Issue & Return Book
- Fine Calculation
- View Issued History
- Search Books

---

## 3. Key Features

### Authentication
- Register
- Login
- Role-based Authorization

### Admin Functionalities
- Add Book
- Update Book
- Delete Book
- View Issued Records
- Manage Users

### Member Functionalities
- Search Book
- Issue Book
- Return Book
- View My Issued Books
- View Fine Details

---

## 4. Backend Architecture

Layered Architecture:

Controller → Service → Repository → Database

OOP Principles Used:
- Encapsulation
- Abstraction
- Inheritance
- Polymorphism

Design Patterns:
- Repository Pattern
- Service Layer Pattern
- DTO Pattern
